# Plan: Biến Rottra thành AI Độc Lập với Qwen 3.5-9B

## Mục tiêu
Rottra hoạt động **hoàn toàn offline** với local LLM, không phụ thuộc cloud APIs (Gemini, Groq, Mistral).

---

## Hiện trạng

### Đã có (hoạt động offline 100%)
- Intent classification (5-tier strategy chain, 200/200 = 100%)
- Hybrid AI engine (`hybrid-ai.ts`): TinyNeuralClassifier + rule engine + 12 persona templates
- Math/algorithm solver
- Greetings & casual Vietnamese fast path
- Semantic cache + TF-IDF local generative model
- RAG reranking (algorithmic)
- Conversation memory (heuristic fallback)
- ONNX sketch recognition (browser)

### Cần LLM để hoạt động tốt
- Natural language chat (trả lời tự nhiên thay vì template)
- Complex query planning (planner.ts)
- Tree-of-Thoughts reasoning (swarm-dispatcher.ts)
- Advanced RAG (HyDE, step-back prompting, query decomposition)
- Creative content generation
- Multi-agent deliberation

### Hiện tại
- `LOCAL_LLM_ENDPOINT` đã có trong code (Priority 0b, thử TRƯỚC cloud)
- `.env` đã set `LOCAL_LLM_ENDPOINT=http://localhost:11434/v1` và `LOCAL_LLM_MODEL=qwen3.5:4b`
- Nhưng **Ollama chưa cài model**, và chưa có Qwen 3.5-9B
- `tiny-llm-runner.ts` đã xóa nhưng import vẫn còn → build error

---

## Kế hoạch thực hiện

### Bước 1: Cài đặt Ollama + Qwen 3.5-9B
- Cài Ollama (nếu chưa có): `curl -fsSL https://ollama.com/install.sh | sh`
- Tải Qwen 3.5-9B Q4_K_M: `ollama pull qwen3.5:9b-q4_K_M`
- Verify: `curl http://localhost:11434/v1/chat/completions -d '{"model":"qwen3.5:9b-q4_K_M","messages":[{"role":"user","content":"Xin chào"}]}'`

### Bước 2: Cấu hình Rottra
**File: `.env`**
- Đổi `LOCAL_LLM_MODEL=qwen3.5:4b` → `LOCAL_LLM_MODEL=qwen3.5:9b-q4_K_M`
- Đảm bảo `LOCAL_LLM_ENDPOINT=http://localhost:11434/v1`

**File: `.env.example`**
- Cập nhật đầy đủ env vars cho người dùng mới

### Bước 3: Tối ưu ai-sdk.ts cho local-first
**File: `src/core/nlp-cognitive/ai-sdk.ts`**
- **Tăng timeout local LLM** từ 180s → 120s (Qwen 3.5-9B inference chậm hơn)
- **Thêm thinking mode support**: Qwen 3.5-9B default thinking mode → cần parse `<think>...</think>` tags
- **Schema self-healing**: Hiện dùng Groq/Mistral cho repair → chuyển sang local LLM
- **Thêm Ollama health check**: Kiểm tra Ollama server availability khi startup

### Bước 4: Xóa/Sửa import lỗi
**File: `src/server/api/agent-router.ts`**
- Tìm và sửa các import từ `tiny-llm-runner.ts` đã xóa
- Loại bỏ các reference đến module không tồn tại

### Bước 5: Tắt cloud APIs (tùy chọn)
**File: `src/core/nlp-cognitive/ai-sdk.ts`**
- Thêm env var `CLOUD_API_ENABLED=false` để tắt hoàn toàn cloud calls
- Khi tắt, pipeline sẽ: Local LLM → CocoLink (nếu có key) → Rule-based fallback
- Cloud APIs vẫn giữ code nhưng không gọi khi `CLOUD_API_ENABLED=false`

### Bước 6: Ollama auto-start (optional)
**File: `src/server/entry.server.ts`**
- Thêm check khi server start: nếu Ollama chưa chạy, log warning
- Không tự start Ollama (để user control)
