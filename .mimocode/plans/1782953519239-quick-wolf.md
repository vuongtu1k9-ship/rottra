## PHASE A: BUG FIXES (5 items, low risk)

### A1. GraphSearchBuffers Race Condition
**File:** `src/core/neural-memory/graph-rag.ts:305-339`
**Problem:** Boolean `lock` is not atomic. Concurrent `semanticBeamSearch` calls share static `Uint8Array`/`Float32Array` buffers ‚Üí data corruption. Under lock contention, falls back to per-call allocation (defeating pooling purpose).
**Fix:** Remove the shared buffer pool entirely. Allocate `visited`, `similarities`, `degrees` arrays per call inside `semanticBeamSearch`. Modern V8 handles short-lived typed arrays efficiently with escape analysis. Delete `GraphSearchBuffers` class. ~20 lines changed.
**Verify:** `tsc --noEmit` passes. No behavioral change.

### A2. swarm-dispatcher nextMarker Ternary
**File:** `src/core/cognitive-swarm/swarm-dispatcher.ts:272`
**Problem:** `nextMarker` is always a truthy string (`=== THOUGHT N ===`). Ternary `nextMarker ? ... : -1` never evaluates false branch. Works by accident (indexOf returns -1 when marker not found).
**Fix:** Replace with `const nextMarker = bestThought < totalThoughts ? \`=== THOUGHT ${bestThought + 1} ===\` : null;`. Then `nextMarker ? branchesText.indexOf(nextMarker) : -1` is correct. Need to pass `totalThoughts` into the function scope. ~5 lines changed.

### A3. 5 Identical Persona Stubs
**File:** `src/core/nlp-cognitive/hybrid-ai.ts:334-453`
**Problem:** `suGia`, `phiAnh`, `bachDiHanh`, `uVuongMau`, `bachLoc` have byte-identical response templates (only character name differs). No unique voice, no domain-specific content.
**Fix:** Rewrite each persona with distinct vocabulary and agricultural personality:
- `suGia` (S·ª≠ Gi·∫£) = scholarly historian, uses classical references, proverb-style wisdom
- `phiAnh` (Phi ·∫¢nh) = young female trader, modern slang, social media savvy
- `bachDiHanh` (B·∫°ch Di H√Ýnh) = wandering merchant, poetic language, travel references
- `uVuongMau` (U V∆∞∆°ng M·∫´u) = queen mother archetype, generous but firm, bulk deals
- `bachLoc` (B·∫°ch L·ªôc) = lucky charm seller, superstitious tone, auspicious numbers
Also fix `phiNguyet` and `nhuNguyet` clones (lines 286-333). ~150 lines rewritten.
